# 自动化控制实验框图存档库

> 面向 **自动化控制课题**，聚焦 **实验硬件搭建、信号连接与信号流程** 的经验整理与技术存档。

---

## 🧭 我在做什么？（一句话）

本仓库整理的 **自动化控制实验框图**，都来自亲身参与过的实验，以及参加学术会议时接触和理解的论文整理。重点关注：

- 实验系统是 **由哪些硬件搭建而成**
- **传感器、执行器、控制器之间的信号如何连接**
- 信号在实验系统中的 **流向、变换与约束关系**

你可以把这里理解为：

> 📂 *一套“看一眼框图 + 看懂信号链，就能快速理解实验结构”的技术档案库*。

---

## 🧠 我的关注点（请务必先看）

> **本仓库不以控制算法推导为核心。不追求极限指标，追求稳定与可复现**

作者背景：**电子专业**， 专注**自动化控制实验方案搭建**，因此每篇文章的重点是：

- ✅ 实验系统的**整体框图结构**
- ✅ 硬件选型逻辑（传感器 / 执行器 / 驱动 / 控制接口）
- ✅ 信号连接关系（模拟 / 数字 / 总线 / 功率链路）
- ✅ 信号流程说明（测量 → 处理 → 控制 → 执行 → 反馈）

❌ 不强调：
- 控制理论证明
- 数学推导过程
- 算法优劣对比

如果你关心的是：

> “**这篇文章里的实验，到底是怎么搭出来的？**”

那这里的内容就是为你准备的。

---

## 🧩 每篇文章都包含什么？

每一篇实验记录，通常包含以下内容：

1. **实验背景说明**  
   
   - 实验目的（验证什么、演示什么）

2. **实验系统整体框图**  
   
   - 由手绘或重绘完成
   - 强调模块划分，而非抽象模型
   - 传感器类型与功能（编码器、位移、力、压力、电流、电压等）  
   
   - 执行器类型与功能（电机、气缸、负载、功率模块等）
   
4. **信号连接说明**（核心内容）  
   
   - 也就是硬件搭建的依据
   
5. **文章编号（唯一标识）**  
   
   - 用于沟通、引用和后续交流

---

## 🔢 关于文章编号（非常重要）

本仓库中的 **每一篇文章标题都包含一个唯一的文章编号**，例如：

- `EXP-ENC-001`
- `EXP-MOTOR-014`
- `EXP-SERVO-CTRL-023`

### 文章编号的作用

- 📎 **快速定位内容**（避免“你说的那个实验是哪一个？”）
- 🤝 **便于沟通合作**（只需回复编号即可）
- 🗂️ **长期技术存档**（防止经验丢失）

> 👉 如果你对某篇实验感兴趣，**联系我时请直接说明文章编号**。
